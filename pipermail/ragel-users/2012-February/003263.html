There is a very useful web site called
http://www.websequencediagrams.com/that allows you to "write" sequence
diagrams in text and generate the
images. Example text:

A->B: text
A-->B: text
A->B: text
activate B
B-->A: text
deactivate B
A->A: text
alt text1
    A->B: text
else text2
    A->B: text
end

Is this text that Ragel would be able to easily parse?
